# Future Ideas

> Long-term ideas that are too early to commit to. Revisit after MVP validation.

---

## Product Ideas

- **Devventory as a CLI tool**: `dv save https://...` — capture from terminal
- **Devventory as a Raycast extension**: Quick capture from anywhere
- **Devventory as a Slack bot**: Save links shared in channels
- **Devventory as a GitHub app**: Save repos, issues, PRs automatically
- **Knowledge marketplace**: Share and discover reference collections
- **Social reading**: Public reading lists, shared annotations

## Technical Ideas

- **Local-first sync**: CRDT-based sync (like Linear, Notion) for offline + real-time
- **WebSocket live updates**: See changes from other sessions instantly
- **P2P sync**: Direct sync between devices without a server
- **Browser extension as P2P relay**: Extensions sync directly
- **Embedded ML model**: On-device AI for privacy and speed

## UX Ideas

- **Spatial navigation**: Move between items like traversing a map (inspired by Apple Vision Pro)
- **Voice capture**: "Hey Devventory, save this page about React Server Components"
- **Infinite canvas**: Zoomable knowledge graph workspace
- **Focus mode**: Full-screen reader with no chrome
- **Daily review**: Tinder-style swipe through "Should I keep or archive this?"

## Out There

- **Devventory as a protocol**: Open standard for knowledge capture, any app can implement
- **Knowledge NFTs**: Own your knowledge graph on-chain (please don't actually do this)
- **AI personal knowledge assistant**: "What did I learn about X last month?"
- **Cross-user knowledge discovery**: "People who saved this also saved..."
