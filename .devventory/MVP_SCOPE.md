# MVP Scope

> The MVP exists to validate one assumption:
>
> **People want one place to capture, organize, and retrieve everything important without remembering where they originally saved it.**

---

## In Scope

### Pages
| Page | Purpose |
|------|---------|
| Dashboard | Continuation page — "What should I continue?" |
| Knowledge | References workspace — three-panel layout for external knowledge |
| Projects | Workspaces that reference knowledge, never duplicate it |
| Radar | Lightweight discovery of relevant resources |

### Features
- Reference capture (URL + metadata + context + notes)
- Folder organization
- Tagging and auto-labels
- Search (Cmd+K global search)
- Inline editing (TipTap)
- Backlinks and related items
- Browser extension (capture URLs + optional context)
- AI enrichment (metadata, summaries, labels)
- Folder tree (hierarchical organization)

### Core Principles
- Capture frictionlessly
- Reveal progressively
- Preserve context (preview panels, sheets, split views)
- Everything is connected (backlinks, related items)
- Build for retrieval (search understands intent)

---

## Out of Scope (MVP)

### Pages Removed
- Prompts (not capture/organization/retrieval)
- Search (merged into Knowledge Cmd+K)
- Chat (AI chat deferred)
- Docs (product documentation deferred)
- Graph (knowledge graph deferred)
- Daily Log (not essential for validation)
- Notes (postponed as "Documents" — roadmap)
- Resources (merged into Knowledge as References)
- Onboarding (simplified to just sign-up flow)

### Features Deferred
- Documents (PDF, images, markdown — roadmap)
- Mobile app (capture-focused, post-MVP)
- Provider-specific workflows (GitHub, YouTube — post-MVP)
- Advanced AI (chat, agents, recommendations)
- Multi-user / team features
- Offline mode
- Plugin system

---

## What We're NOT Building (MVP)

- Notion clone
- Obsidian clone
- Google Drive clone
- WhatsApp clone
- Readwise clone
- Jira clone

We're building **the fastest way to capture knowledge with context**.
